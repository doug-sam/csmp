第二章 登录

功能概述  

系统用户种类请参见第一章，其中一线登录系统主要功能是创建报修信息。MVS用户登录系统主要功能是运用、统计、管理系统中数据，作为系统的核心管理者。其它用户登录系统，如客户、二线、现场工程师登录系统，其主要功能是查阅系统中数据。系统中的原始角色是管理员，由管理员添加各种身份的用户。有效的登录时间为一天，被禁用的用户和无权限的用户将无法登录。

B. 操作说明  

       进入MVS客户服务管理系统，默认进入登录页面，其登录界面如【图2-1】所示:

【图2-1】
以正确用户登录系统后，可以根据用户身份角色和权限对系统进行操作。当不再使用系统时，请占击右上方“注销”，可以确保在本地机器上必需使用帐号密码和验证码才能再次进入系统，验证码的有效时间为登录页面后的20分钟内，超时后验证码讲失效，需要通过刷新页面来跟新验证码。


第三章：报修管理

总体介绍

报修管理模块主要记录了在不同处理进度下的报修信息【图3-1】所示：

功能总览

  • 新建报修:可以新建出一条call。
  • 批量报修:可以新建出多条call。注意：只能将call分派给和登录用户相同工作组的二线。
  • 等待安排上门:记录二线处理方法为上门处理并且正在等待备件的call。
  • 未处理:可以对未处理的call进行处理，转派，查看最近三次报修，编辑报修信息，店铺催促，查看店铺报修历史和对未处理的call进行搜索。
  • 处理中:可以对处理中的call进行处理，转派，查看最近三次报修，编辑报修信息，店铺催促，查看店铺报修历史和对处理中的call进行搜索。
  • 已完成:可以查看处理记录，转派,进行回访，查看最近三次报修，编辑报修信息，查看店铺报修历史,关闭call和对已完成的call进行搜索。
  • 已关闭:可以查看处理记录，转派，进行回访，查看最近三次报修，编辑报修信息，查看店铺报修历史,关闭call，对已关闭的call进行搜索。点击处理记录，可以查看处理过程并且能对操作进行回滚。
  • 报修查询:可以查看处理记录，查看最近三次报修，编辑报修信息，查看店铺报修历史,回滚操作，对所有状态的call进行搜索。

使用方法

  • 新建报修:填写必要的店铺信息和报修信息后点击保存即可新建出一条call。


注意：报修时间是可以修改的，填写部分店铺编号就会在文本框下方显示预选值，点击       预选值就会自动将预选值填写进文本框：  

填写好店铺编号后点击确定会自动填写店铺所属的客户、品牌、省份、城市、店铺名称、    电话、地址，确定键也可以用快捷键Alt+a代替：

只能将call分派给和登录用户相同工作组的二线；SLA扩展是超市的自定义表达式；    重置按钮可以将所有信息设置为默认值。

  • 批量报修:首先通过特定条件搜索出需要建call的店铺并勾选，再填写必要的报修信息，点击保存即可以新建出多条call。


注意：批量报修只限于单一客户。

  • 等待安排上门:安排上门：点击安排上门，然后填写上门工程师等必要信息后提交即可。

处理记录：点击处理记录，可以对操作进行回滚。

报修信息：查看所有报修历史：点击查看所有报修历史：
会自动跳转到报修查询并会显示该店铺的所有call。店铺催促：点击店铺催促：     填写好催促信息然后提交：   
店铺催促会在以下地方显示：

店铺号：点击店铺号
注意：预约上门时间是可以修改的，上门工程师可以选择当前工程师上门或转派第三方,
当前工程师：和当前登录用户相同工作组的现场工程师，转派第三方：将call转派给其 他工作组的现场工程师，操作回滚只能回滚到第一步。

  • 未处理:点击前往处理,若已解决则勾选已解决并填写过程备注和解决方案提交即可，若未解决则勾选未解决并填写过程备注，提交后再选择处理方法即可。点击转派，选择好二线并按确定即可完成call的转派，注意：只能转派给相同工作组的二线，系统单号由年月日时加毫秒三位数组成，SLA为call的超时时限，SLA倒计时由SLA开始倒数若时限小于0即为超时,SLA只能通过刷新网页更新
  • 处理中:点击前往处理,若以解决则勾选已解决并填写过程备注和解决方案提交即可，若未解决则勾选未解决并填写过程备注，提交后再选择处理方法即可。点击转派，选择好二线并按确定即可完成call的转派，注意只能转派给相同工作组的二线。选择好搜索条件点击搜索即可，SLA只能通过刷新网页更新。
  • 已完成:点击转派，选择好二线并按确定即可完成call的转派，注意只能转派给相同工作组的二线。点击回访，可以对报修处理进行评价，填写完必要信息提交即可。选择好搜索条件点击搜索即可。通常call是由二线完成的。注意：只能回滚到第一步，处理时间等于执行操作的时间，记录时间等于添加记录的时间，SLA只能通过刷新网页更新。
  • 已关闭:选择好搜索条件点击搜索即可。通常call是由一线关闭的。注意：已关闭的call不能进行回滚，只能查看转派记录不能进行转派，SLA只能通过刷新网页更新。
  • 报修查询:选择好搜索条件点击搜索即可。

第四章：系统管理

总体介绍

系统管理模块主要提供了系统日志的查看，邮箱和短信的发送和设置，还可以对系统参数进行设置和查看。【图4-1】所示：

 

 

 

 

 

 

 


【图4-1】

功能总览

 

  • 系统日志:记录系统出错的日志，日志信息有编号、所属分类、标题、引错用户、添加日期和系统出错日期，并且可以在查看中查看日志的详细信息。还可以通过选择类型来进行该类型的日志搜索,还有可以勾选若干条日志进行删除操作。
  • 报修回滚日志:记录报修单回滚操作的日志，日志信息有序号、操作人、操作时间和操作报修单，可以勾选若干条日志进行删除操作。
  • 数据导入:只能从2003版本的excel文件中进行中类数据导入、故障类整体导入、小类数据导入和店铺信息导入，还可以进行中类导入模版、店铺导入模版和故障类整体导入模版的下载。注意：中类数据导入：1.数据中不能出现空行，否则空行以下的数据将不被导入。系统中空行表示结束 2.模板中的表头，工作簿名不能修改 3.每列的顺序不能修改 4.一级故障名或一级故障ID只需填写任意一列 5.如系统在导入过程中出错。将会事务回滚，没有记录会插入到数据库中；店铺信息导入：1.数据中不能出现空行，否则空行以下的数据将不被导入。系统中空行表示结束 2.模板中的表头，工作簿名不能修改 3.每列的顺序不能修改 4.对应省份，对应品牌，所属省份，所属城市必需在系统中存在，其中“对应品牌”必需严格一致。“省份”，”城市“中不能出现空格，省，市二字可有可无。 5.如系统在导入过程中出错。将会事务回滚，没有记录会插入到数据库中。
  • 邮箱设置:可以进行系统邮箱的设置。注意：host：主机；Port：端口；Email Address：邮箱地址；Email Pwd：邮箱密码；Disply Name：显示姓名；Reply Address：回复地址；Reply Name：回复人姓名。
  • 发邮件:可以用“邮箱设置”里设置好的邮箱地址密码等信息发送邮件，并且能测试“邮箱设置”是否正确。
  • 短信设置:可以对短信模版进行修改。注意：host：主机地址；Port：端口；accountID：账户；password：密码；serviceID：服务，若已勾选总开关，在处理方式为上门解决的call选择好上门工程师后会出现一个发送短信的按钮，若总开关没有勾选，此按钮将会被隐藏，短信模版的关键字是可以删除的，若没删除关键字将会返回相应的数据。
  • 发短信:可以进行系统短信的发送，并且能测试“短信设置”是否正确。
  • 邮件模版:可以对邮件模版进行修改。注意：邮箱模版的关键字是可以删除的，若没删除关键字将会返回相应的数据。
  • 邮件接口:通过邮件做接口同步和yum客户的报修数据。注意：总体设置：接口开关是用来控制是否发开邮件接口。邮箱设置：host：主机；Port：端口；Email Address：邮箱地址；Email Pwd：邮箱密码；Disply Name：显示姓名；Reply Address：回复地址；Reply Name：回复人姓名；对方接口邮箱：YUM做邮件接口的邮箱。邮件模版一、二、三分别是三种不同的接口邮件的模版，邮箱模版的关键字是可以删除的，若没删除关键字将会返回相应的数据。
  • 参数设置:可以进行系统参数的设置。
  • 系统信息:可以查看系统的信息。

使用方法

  • 系统日志:可以通过日志类型搜索并查看系统日志，并可以通过勾选日志进行批量删除系统日志。
  • 报修回滚日志:可以查看报修回滚日志，并可以通过勾选日志进行批量删除报修回滚日志。
  • 数据导入:选择需要导入的数据类型，并选择需要进行导入的文件进行导入。
  • 邮箱设置:对系统邮箱的设置进行修改。
  • 发邮件:填写完收件人，主题，内容后点击发送即可发送邮件。
  • 短信设置:对系统短信设置和系统短信模版进行修改。
  • 发短信:填写好对方手机号码和短信内容后点击提交即可发送短信。
  • 邮件模版:对邮件模版进行修改。
  • 邮件接口:进入总体设置勾选接口开关开启邮箱接口，设置yum系统id。进入邮箱设置设置yum接口邮箱等信息。进入模版一、二、三对接口邮件模版进行修改。
  • 参数设置:对系统参数进行修改然后提交。
  • 系统信息:查看服务器的信息。

 第五章：基础信息管理

 总体介绍

 基础信息管理管理模块主要对客户、品牌、店铺、省份、城市、大中小类故障、解决方案进行增加、修改、删除操作，并且可以通过特定的条件进行搜索。还可以初始化所有项目故障类。【图5-1】所示：


 

 

 

 

 

 

 

 


 功能总览

  • 客户管理:可以搜索、增加、修改和删除客户和初始化该客户大中小类的项目故障类。
  • 品牌管理:可以搜索、增加、修改和删除品牌。
  • 店铺管理:可以搜索、增加、修改和删除店铺。
  • 省份管理:可以增加和修改省份。
  • 城市管理:可以搜索、增加、修改和删除城市。
  • 大类故障管理:可以搜索、增加、修改和删除大类故障。
  • 中类故障鼓励:可以搜索、增加、修改和删除中类故障。
  • 小类故障管理:可以增加、修改和删除小类故障。
  • 初始化所有项目故障类:建立名称为项目的大中小类故障。
  • 解决方案管理:可以搜索、修改和删除解决方案。

 

 

 使用方法

  • 客户管理:添加客户：点击添加，填写好用户的名称、联系人、联系电话、电子邮件、所属城市、勾选是否使用，若勾选使用则使客户生效，若勾选停用则使客户失效，点击提交即可完成添加客户。编辑客户：点击编辑可以修改用户的名称、联系人、联系电话、电子邮件、所属城市、勾选是否使用，若勾选使用则使客户生效，若勾选停用则使客户失效，点击提交即可完成编辑客户，初始化该客户大中小类的项目故障类作用是建立名称为该客户项目的大中小类故障。删除客户：在勾选框勾选中需要删除的客户，然后点击删除选中即可删除客户。搜索客户：在搜索栏输入客户名，然后点击搜索就可以完成客户搜索。
  • 品牌管理:添加品牌：点击添加，填写好品牌的名称、所属客户、联系人、联系电话、电子邮件、勾选是否使用，若勾选使用则使品牌生效，若勾选停用则使品牌失效，点击提交即可完成添加品牌。编辑品牌：点击编辑可以修改品牌的名称、所属客户、联系人、联系电话、电子邮件、勾选是否使用，若勾选使用则使品牌生效，若勾选停用则使品牌失效，点击提交即可完成编辑品牌。删除品牌：在勾选框勾选中需要删除的品牌，然后点击删除选中即可删除品牌。搜索品牌：在搜索栏输入品牌名，所属客户，然后点击搜索就可以完成品牌搜索。
  • 店铺管理:添加店铺：点击添加，填写好店铺编号、店铺名称、所属客户、所属品牌、所属地区、地址、电话、勾选是否使用，若勾选使用则使店铺生效，若勾选停用则使店铺失效，点击提交即可完成添加店铺。编辑店铺：点击编辑可以修改店铺编号、店铺名称、所属客户、所属品牌、所属地区、地址、电话、勾选是否使用，若勾选使用则使店铺生效，若勾选停用则使店铺失效，点击提交即可完成修改店铺。删除店铺：在勾选框勾选中需要删除的店铺，然后点击删除选中即可删除店铺。搜索店铺：在搜索栏输入所属客户，所属品牌，所属省份，所属城市，店铺名或编号，店铺电话，然后点击搜索就可以完成店铺搜索。
  • 省份管理:添加省份：点击添加，填写好名称，勾选是否启用，若勾选中使用则使省份生效，若没勾选中启用则使省份失效，点击提交即可完成添加省份。编辑省份：点击笔状的图标，修改名称，勾选是否启用，若勾选中使用则使省份生效，若没勾选中启用则使省份失效，点击提交即可完成修改省份。
  • 城市管理:添加城市：点击添加，填写好名称，所属省份，勾选是否启用，若勾选中使用则使城市生效，若没勾选中启用则使城市失效，点击提交即可完成添加城市。编辑省份：点击笔状的图标，修改名称，所属省份，勾选是否启用，若勾选中使用则使城市生效，若没勾选中启用则使城市失效，点击提交即可完成修改城市。搜索城市：在搜索栏输入所属省份，城市名，然后点击搜索就可以完成城市搜索。
  • 大类故障管理:添加大类故障：点击添加，填写好名称、所属客户，勾选是否使用，若勾选使用则使大类故障生效，若勾选停用则使大类故障失效，点击提交即可完成添加大类故障。修改大类故障：点击编辑，修改名称、所属客户，勾选是否使用，若勾选使用则使大类故障生效，若勾选停用则使大类故障失效，点击提交即可完成修改大类故障。删除大类故障：在勾选框勾选中需要删除的大类故障，然后点击删除选中即可删除大类故障。搜索大类故障：在搜索栏输入大类故障名，所属客户，然后点击搜索就可以完成大类故障搜索。
  • 中类故障管理:添加中类故障：点击添加，填写好名称、所属客户，所属大类，勾选是否使用，若勾选使用则使中类故障生效，若勾选停用则使中类故障失效，点击提交即可完成添加中类故障。修改中类故障：点击编辑，修改名称、所属客户，所属大类，勾选是否使用，若勾选使用则使中类故障生效，若勾选停用则使中类故障失效，点击提交即可完成修改中类故障。删除中类故障：在勾选框勾选中需要删除的中类故障，然后点击删除选中即可删除中类故障。搜索中类故障：在搜索栏输入所属客户，所属大类，中类故障名，然后点击搜索就可以完成中类故障搜索。
  • 小类故障管理:添加小类故障：点击添加，填写好名称、所属客户，所属大类，所属中类，优先级，SLA，勾选是否使用，若勾选使用则使小类故障生效，若勾选停用则使小类故障失效，点击提交即可完成添加小类故障。修改小类故障：点击编辑，修改名称、所属客户，所属大类，所属中类，优先级，SLA，勾选是否使用，若勾选使用则使小类故障生效，若勾选停用则使小类故障失效，点击提交即可完成修改小类故障。删除小类故障：在勾选框勾选中需要删除的小类故障，然后点击删除选中即可删除小类故障。搜索小类故障：在搜索栏输入所属客户，所属大类，所属中类，小类故障名，然后点击搜索就可以完成小类故障搜索。
  • 初始化所有项目故障类:自动建立名称为项目的大中小类故障。注意：如果已存在则不再创建。
  • 解决方案管理:修改解决方案：点击编辑，修改方案名，解决次数，点击提交即可完成修改解决方案。删除解决方案：在勾选框勾选中需要删除的解决方案，然后点击删除选中即可删除解决方案。搜索解决方案：在搜索栏输入所属品牌，所属故障类，然后点击搜索就可以完成解决方案搜索。
